# Known Limitations

KAIROS v0.1 is an experimental manual-first kit.

It is not a fully automated AI operating system yet.

---

## Current Limitations

### No automatic AI execution

KAIROS does not currently call ChatGPT, Claude, Gemini, Grok, Kimi, Codex, or Jules directly.

It only recommends routing and prompts.

---

### No real n8n workflow execution

n8n workflow files are currently placeholders or planning templates.

Real importable workflows are planned for future versions.

---

### No Hermes bot yet

Hermes is currently a design concept and role definition.

It does not yet send Slack, Telegram, Discord, or email messages.

---

### No GUI console yet

KAIROS is currently operated through:

- PowerShell scripts
- Obsidian
- Manual review

A GUI console is planned.

---

### No Local LLM classification yet

Local LLM is currently part of the architecture and role model.

It does not yet classify tasks or assets automatically.

---

### Manual approval required

High-impact actions should remain manual.

This is intentional.

KAIROS prioritizes user control over premature automation.

---

### Windows-first scripts

Current scripts are PowerShell-based and tested primarily on Windows.

Cross-platform support is planned.

---

## Design Choice

These limitations are intentional for v0.1.

KAIROS starts as a safe, local-first, human-controlled operations kit before becoming more automated.
